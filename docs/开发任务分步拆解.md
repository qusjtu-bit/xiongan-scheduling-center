## Step 1：基础底座搭建

**目标**：搭好数据库和认证网关骨架，为后续所有模块提供基础设施。

### 1.1 数据库初始化
- 创建达梦/MySQL数据库 `xiongan_jjzx`
- 建表：`sys_user`、`sys_dept`、`sys_role`、`sys_permission`、`sys_user_role`、`sys_role_permission`
- 初始化基础数据：8个处室部门、5个角色（局领导/处长/经办人/管理员/访客）、默认权限

### 1.2 统一身份认证服务（auth-service）
- JWT Token 签发与校验
- 登录接口 `POST /api/auth/login`（账号密码）
- Token刷新接口 `POST /api/auth/refresh`
- 当前用户信息接口 `GET /api/auth/userinfo`
- 权限校验注解 `@RequirePermission`

### 1.3 API网关（gateway-service）
- 路由转发（统一前缀 `/api/**`）
- JWT鉴权拦截
- 接口限流（令牌桶）
- 请求日志记录

### 1.4 用户与组织管理（user-service）
- 用户CRUD `GET/POST/PUT/DELETE /api/users`
- 部门CRUD `GET/POST/PUT/DELETE /api/depts`
- 角色CRUD `GET/POST/PUT/DELETE /api/roles`
- 权限分配 `PUT /api/roles/{id}/permissions`

### 验收标准
- [ ] 能用默认账号登录，获取JWT Token
- [ ] Token过期后能自动刷新
- [ ] 无效Token请求被网关拦截返回401
- [ ] 用户、部门、角色、权限CRUD全流程跑通

## Step 2：统一工作门户

**目标**：全局人员通过一个入口登录，看到个性化工作台和所有系统导航。

### 2.1 登录页面
- 科技感UI（深色主题、渐变背景）
- 账号密码登录 → 对接 auth-service
- 登录成功后跳转主界面

### 2.2 主界面框架
- 顶部导航栏（Logo + 用户信息 + 消息铃铛 + 退出）
- 左侧菜单树（按业务领域分组）
- 右侧内容区（动态加载各模块页面）
- 响应式布局（适配1920×1080大屏）

### 2.3 统一工作台首页
- 待办事项卡片（聚合各系统待办数量）
- 预警消息卡片（最新预警TOP5）
- 快捷入口（常用系统一键直达）
- 工作统计（本月办件量、审批时效）

### 2.4 应用导航
- 应用菜单管理（后台维护可访问系统列表）
- 按角色展示不同菜单
- 支持搜索、收藏、最近使用

### 2.5 消息中心（message-service）
- 站内消息列表 `GET /api/messages`
- 消息已读/删除 `PUT /api/messages/{id}/read`
- 待办任务列表 `GET /api/todos`
- 待办处理跳转

### 验收标准
- [ ] 登录页科技感、响应正常
- [ ] 登录后进入工作台，看到个性化数据
- [ ] 菜单按角色权限动态显示
- [ ] 待办和消息列表数据正确
- [ ] 切换不同模块页面加载正常

## Step 3：建交数据中枢

**目标**：把17个业务系统的数据归集起来，形成四大主题库，提供统一数据API。

### 3.1 数据归集管道
- 数据源配置管理（配置各业务系统数据库连接）
- 定时批量同步（DataX / 自研同步任务）
- 实时增量同步（CDC监听binlog）
- 同步任务监控面板

### 3.2 四大主题数据库
- **城乡建设主题库**：项目、企业、人员、工地、证书
- **交通运输主题库**：路网、车辆、线路、枢纽、事件
- **水利水务主题库**：河湖、管网、水站、工情
- **城市管理主题库**：市政设施、管廊、环卫、园林、事件

### 3.3 数据治理
- 数据清洗规则配置（空值、格式、去重）
- 数据标准化（统一编码、统一字典）
- 数据质量稽核报告

### 3.4 数据资源目录
- 资源编目管理 `GET/POST/PUT/DELETE /api/data-resources`
- 资源分类浏览（按领域、按系统）
- 资源详情查看（字段定义、数据示例、更新频率）

### 3.5 数据共享API
- 通用数据查询接口 `GET /api/data/{resource_code}`
- 按条件检索 `POST /api/data/{resource_code}/search`
- 数据导出 `GET /api/data/{resource_code}/export`

### 3.6 标准指标体系
- 指标定义管理 `GET/POST/PUT /api/indicators`
- 指标数据查询 `GET /api/indicators/{code}/data`
- 指标定时计算任务（Cron调度）

### 验收标准
- [ ] 至少能配置并同步2个模拟数据源
- [ ] 四大主题库表结构创建完成，有Mock数据
- [ ] 数据资源目录可浏览、可检索
- [ ] 通用数据查询API可用
- [ ] 标准指标定义≥50项，数据可查

## Step 4：全局态势感知大屏

**目标**：局领导打开大屏，一屏看到全局核心运行态势。

### 4.1 大屏布局
- 顶部指标条（在建项目数、今日办件量、预警数、办结率、在线率）
- 中央CIM三维底图区
- 左侧四大行业指标卡片
- 右侧项目进度TOP10 + 预警列表 + 审批趋势
- 底部事件滚动播报

### 4.2 CIM底图对接
- Cesium初始化，加载雄安CIM平台WMTS底图
- 坐标系转换（雄安城市坐标系 ↔ WGS84）
- 地图基础操作（缩放、平移、旋转、全屏）

### 4.3 业务图层叠加
- 工程项目图层（点标注 + 状态色分类）
- 在建工地图层（面标注 + 安全等级）
- 综合管廊图层（线标注）
- 河湖水系图层（面标注 + 水质色）
- 预警事件图层（动态闪烁）
- 图层开关控制面板

### 4.4 数据可视化图表
- ECharts柱状图（各片区项目分布）
- ECharts饼图（事项办理状态占比）
- ECharts折线图（审批时效趋势）
- ECharts雷达图（四大行业综合评分）
- 数字翻牌动画效果

### 4.5 实时数据刷新
- WebSocket推送预警事件
- 定时轮询刷新指标数据（30秒间隔）
- 事件列表自动滚动

### 验收标准
- [ ] 大屏整体布局科技感、不花哨
- [ ] CIM底图正常加载（或用底图占位图替代）
- [ ] 各图表数据正确、动画流畅
- [ ] 图层开关可控制显隐
- [ ] 预警事件实时推送可见

## Step 5：规建管一体化

**目标**：工程项目从立项到运营，全流程可视化、可追溯、可管控。

### 5.1 项目列表页
- 表格展示项目列表（名称、类型、片区、建设单位、当前阶段、状态）
- 多条件检索（名称搜索、类型筛选、片区筛选、状态筛选）
- 分页、排序
- 导出Excel

### 5.2 项目详情页
- 基本信息卡（投资额、建筑面积、建设单位、开工日期等）
- 空间位置地图标注（小地图）
- 全流程时间轴展示

### 5.3 全流程时间轴
- 六阶段时间轴（立项→规划→审批→建设→竣工验收→运营维护）
- 每个节点显示：办理人、办理部门、起止时间、办理时长、办理结果
- 超时节点红色高亮、补正节点橙色标注
- 点击节点查看详情（审批材料、办理结果、验收报告）

### 5.4 项目对比分析
- 选择2-5个项目进行横向对比
- 进度对比图（各阶段耗时柱状图）
- 质量对比图（隐患数/整改率）
- 安全对比图（违规次数/覆盖率）

### 5.5 异常预警
- 审批超时自动预警
- 进度滞后自动预警
- 隐患未整改自动预警
- 预警推送到消息中心

### 验收标准
- [ ] 项目列表检索、分页、导出正常
- [ ] 时间轴完整展示六阶段，超时节点高亮
- [ ] 对比分析图表数据正确
- [ ] 预警规则可配置，触发后消息可见

## Step 6：业务专题一张图

**目标**：每个处室有自己的专题页，看本处室业务全貌。

### 6.1 专题切换框架
- 顶部Tab切换：城乡建设 | 交通运输 | 水利水务 | 城市管理
- 每个专题独立页面布局
- 左侧导航（业务总览/明细/协同/台账）

### 6.2 城乡建设专题
- 业务总览（在建项目数、施工许可数、消防验收率、造价发布数）
- 工程监管（项目分布图、隐患统计、整改闭环率）
- 建筑市场（企业资质统计、实名制考勤率）
- 绿色建筑（绿建项目数、装配式占比、能耗监测）
- 农房建设（建设进度、危房改造完成率）
- 台账管理（建筑垃圾备案、运输车辆、造价信息、档案验收台账）

### 6.3 交通运输专题
- 路网运行（实时路况、拥堵指数、事故分布）
- 运输监管（企业统计、营运车辆、非法营运查处）
- 公交出行（线路覆盖、站点分布、客流趋势、准点率）
- 水运安全（白洋淀船舶动态、港口岸线）
- 交通枢纽（雄安站运行、客流、接驳）

### 6.4 水利水务专题
- 河湖治理（河湖分布、水质监测点、达标率、河湖长巡查）
- 防汛排涝（水位监测、雨量监测、泵站运行）
- 供水排水（管网分布、水质监测、污水厂运行）
- 水资源管理（用水总量、地下水、节水指标）
- 水土保持（流失分布、治理进度）

### 6.5 城市管理专题
- 市容环卫（环卫覆盖图、垃圾分类点、清运量、洁净度）
- 市政设施（道路桥梁、路灯亮灯率、井盖状态）
- 综合管廊（运营里程、入廊管线、环境监测、运维工单）
- 供热燃气（管网覆盖、热源运行、燃气监测、报修）
- 园林绿化（绿地分布、公园管理、古树保护、绿化覆盖率）

### 6.6 专题交互与导出
- 多条件检索（项目名称、处室、时间、片区）
- 维度筛选、数据逐层下钻
- PDF/Excel一键导出

### 验收标准
- [ ] 四大专题Tab切换正常
- [ ] 每个专题的5-6个子模块数据完整
- [ ] 图表展示正确、可交互
- [ ] 检索和导出功能可用

## Step 7：专属智能体

**目标**：AI赋能业务，实现预警、分析、决策、处置智能化。

### 7.1 智能预警智能体
- 工程建设预警（实名制+考勤+参保+工资关联分析）
- 施工安全预警（IoT感知数据阈值判断）
- 交通运输预警（路网流量异常检测）
- 水利水务预警（水位/雨量阈值）
- 城市管理预警（管廊环境/燃气/供热异常）
- 预警分级（一般/较大/重大/特大）+ 自动推送

### 7.2 智能分析智能体
- 按月自动统计核心指标
- 生成多维度分析报告（老旧小区改造、隐患整改、造价走势、垃圾分类、路网养护、河湖水质）
- 报告模板自动填充 + 自然语言摘要

### 7.3 智能决策智能体
- 农村危房改造决策模型（排查数据+改造进度+政策要求→督办建议）
- 公交线路优化决策模型（客流OD+线路覆盖→调整建议）
- 防汛调度决策模型（水位预报+排涝能力→调度方案）

### 7.4 智能处置智能体
- 建筑垃圾处理方案备案自动化初审
- 运输车辆备案自动化初审
- 民用建筑节能验收备案自动化初审
- 人防简易备案自动化初审
- 材料完整性校验 → 表单规范性校验 → 要件一致性比对 → 流转/补正

### 7.5 AI技术底座
- 大模型接入（政策问答、报告生成）
- 知识库构建（政策法规库、业务知识库、历史案例库）
- RAG检索增强生成
- 预测模型训练（时序预测、异常检测）

### 验收标准
- [ ] 五大领域预警模型至少3个可演示
- [ ] 月度分析报告可自动生成
- [ ] 至少1个决策模型输出可落地的建议
- [ ] 至少1类备案业务实现自动化初审
